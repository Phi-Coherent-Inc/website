phi::sign · SLH-DSA/XMSS family · stateful

φSign — hash-based signature

Hypertree of L(2)=3 layers × height L(4)=7, WOTS width F(7)=13, FORS F(7)=13 trees. Security rests only on φHash preimage/collision resistance — no elliptic curves, no algebraic hardness. Genuinely slow here: this build is single-threaded (GitHub Pages can't serve the COOP/COEP headers multi-threaded WASM needs), so the hypertree's one-time-key fan-out runs serially — keygen ≈20s, sign ≈60s. Both run in a Worker so the page stays responsive; verify is fast (≈0.3s).

φKEM — key encapsulation (two-party)

Alice publishes a key. Bob encapsulates a fresh shared secret to it — and, separately, encrypts a message directly to it (φKEM + φHash-derived DEM key + φCipher AEAD). Alice recovers both. All keygen/encaps/decaps calls are millisecond-scale.
phi::pqc / phi-lattice · φDSA (ML-DSA-87 class) · Module-LWE/SIS, stateless

φDSA — lattice signature

Fiat–Shamir with aborts, stateless — no counter to persist, the stateless complement to φSign. Independent assumption from φSign (lattices, not hashes); the hybrid signature (not exposed in this demo) binds both.

Reading the hash-based rows. φHash's output width sets the classical search space directly; the post-Grover (quantum) figure is half the classical bit count — the standard quadratic Grover speedup, computed live from the actual digest length above, not a hardcoded table. φCipher's key and φSign's public-key commitment are the same φHash output at the same tier, so they inherit the identical bit strength. Tier 0 (116-bit quantum) is NIST Category 3; tier 2 (304-bit quantum) exceeds NIST Category 5, whose quantum floor is 232 bits.
φKEM (φ-KEM-304). Module-LWE, n=512, module rank k=3, q=3329, η=3 (CBD noise). Sized so its estimated quantum core-SVP (β≈1462 → ≈427-bit classical, ≈387-bit quantum) clears both the suite's own 304-bit floor and NIST Category 5's 232-bit quantum bar. The one φ-native load-bearing element is the woven Reed–Muller reconciliation (RM(1,3)) inside encode/decode — the hardness itself is plain Module-LWE.
φDSA. Fiat–Shamir with aborts (the ML-DSA algorithm) over the suite's shared proof-system field (N=256, Q=1342832641, k=l=9, η=7, τ=60) — one field across the proof system and the signature, so signatures can be verified inside recursive proofs natively. The parameter set is derived from first principles and certified to dominate ML-DSA-87 — NIST FIPS-204's own Category 5 set — on every axis under the same estimator model (certified ≈255-bit quantum forgery floor, clearing Category 5's 232-bit bar), with the certification in the regime where the conservative bound is provable. Not byte-conformant FIPS-204 (own encodings, validated against its own KATs, not ACVP) — the security margin is certified above the standards one; the wire format is the suite's own.
